\capitulo{2}{Objetivos del proyecto}

El objetivo principal consiste en diseñar, desplegar y validar en Microsoft Azure un servicio que automatice el registro y la clasificación de solicitudes operativas de TI sin almacenar credenciales sensibles en el código.

Los objetivos específicos que concretan este propósito son los siguientes:

\begin{itemize}
    \item Analizar las ventajas de las plataformas cloud en entornos empresariales.
    \item Diseñar una arquitectura segura basada en servicios cloud.
    \item Implementar una API REST utilizando Python y Flask.
    \item Desplegar la aplicación en Microsoft Azure.
    \item Gestionar secretos y configuraciones sensibles mediante Azure Key Vault.
    \item Automatizar el despliegue de la aplicación mediante scripts reproducibles con Azure CLI.
    \item Gestionar el proyecto utilizando metodologías ágiles y control de versiones.
    \item Implementar una clasificación automática y explicable mediante reglas de negocio.
    \item Analizar la calidad del código utilizando SonarQube/SonarCloud.
    \item Generar documentación técnica y funcional del sistema.
\end{itemize}

Estos objetivos conectan materias del grado que en el prototipo aparecen relacionadas: ingeniería del software para requisitos y pruebas, seguridad para secretos y permisos, sistemas distribuidos para la separación de servicios y gestión de proyectos para la planificación por iteraciones.
