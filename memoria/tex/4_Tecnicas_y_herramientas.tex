\capitulo{4}{Técnicas y herramientas}

Durante el desarrollo del proyecto se han utilizado diversas metodologías y herramientas que han permitido gestionar el proceso de desarrollo de forma estructurada.

\section{Metodología de desarrollo}

El proyecto se ha desarrollado siguiendo principios de metodologías ágiles, organizando el trabajo en diferentes iteraciones o sprints en la herramienta de Zube. Para el seguimiento de las tareas se ha utilizado el sistema de gestión de incidencias de GitHub.

\section{Control de versiones}

El control de versiones del proyecto se ha gestionado mediante Git, utilizando la plataforma GitHub como repositorio central del código fuente y la documentación.

\section{Plataforma cloud}

El despliegue del sistema se realiza utilizando la plataforma Microsoft Azure, que permite gestionar la infraestructura necesaria para la ejecución de la aplicación.

\section{Integración continua}

Se ha configurado un sistema de integración continua mediante GitHub Actions que permite automatizar tareas de compilación y verificación del código.

\section{Calidad del código}

Para evaluar la calidad del código desarrollado se ha utilizado la herramienta SonarQube, que permite analizar aspectos como la complejidad del código, la duplicidad o posibles vulnerabilidades de seguridad.
