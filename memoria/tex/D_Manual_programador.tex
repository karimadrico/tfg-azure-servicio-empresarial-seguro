\apendice{Documentación técnica de programación}

\section{Introducción}

Este anexo describe la estructura técnica del proyecto y los pasos necesarios para ejecutar y mantener la aplicación.

\section{Estructura de directorios}

La estructura principal del repositorio es la siguiente:

\begin{itemize}
    \item app.py: API principal desarrollada en Flask.
    \item requirements.txt: Dependencias Python.
    \item .github/workflows: Configuración CI/CD.
    \item docs/: Documentación técnica.
    \item tex/: Memoria y anexos en LaTeX.
\end{itemize}

\section{Manual del programador}

Para ejecutar el proyecto localmente es necesario:

\begin{enumerate}
    \item Instalar Python.
    \item Instalar dependencias mediante pip.
    \item Ejecutar el archivo app.py.
\end{enumerate}

Comando de instalación:

\begin{verbatim}
pip install -r requirements.txt
\end{verbatim}

Ejecución del sistema:

\begin{verbatim}
python app.py
\end{verbatim}

\section{Compilación, instalación y ejecución del proyecto}

La aplicación puede desplegarse automáticamente utilizando GitHub Actions y Azure App Service.

\section{Pruebas del sistema}

Las pruebas realizadas incluyen:

\begin{itemize}
    \item Verificación de endpoints.
    \item Validación de respuestas JSON.
    \item Validación de despliegue.
    \item Verificación de monitorización.
\end{itemize}
