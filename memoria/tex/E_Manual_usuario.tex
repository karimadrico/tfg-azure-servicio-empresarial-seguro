\apendice{Documentación de usuario}

\section{Introducción}

Este anexo describe cómo utilizar el sistema de gestión de incidencias empresariales desplegado en Microsoft Azure.

\section{Requisitos}

El usuario necesita:

\begin{itemize}
    \item Cliente HTTP (navegador, Postman o curl).
    \item Conexión a Internet.
    \item Token de autenticación Bearer (excepto en entorno local sin clave configurada).
\end{itemize}

\section{Acceso al servicio}

La API está desplegada en Azure App Service:

\begin{verbatim}
https://app-tfg-incidencias-dev.azurewebsites.net
\end{verbatim}

También puede registrarse una incidencia mediante la Logic App, que reenvía la petición automáticamente a la API.

\section{Operaciones disponibles}

\subsection{Consultar estado del servicio}

\begin{verbatim}
GET /
\end{verbatim}

\subsection{Crear incidencia}

\begin{verbatim}
POST /incidencias
Authorization: Bearer <token>
Content-Type: application/json

{
  "titulo": "Error servidor",
  "descripcion": "El servidor principal no responde",
  "reportado_por": "usuario@empresa.com",
  "prioridad": "alta"
}
\end{verbatim}

El sistema clasificará automáticamente la incidencia y devolverá prioridad, categoría y nivel de confianza.

\subsection{Consultar incidencias}

\begin{verbatim}
GET /incidencias?estado=abierta&prioridad=alta
Authorization: Bearer <token>
\end{verbatim}

\subsection{Consultar métricas}

\begin{verbatim}
GET /metricas
Authorization: Bearer <token>
\end{verbatim}
