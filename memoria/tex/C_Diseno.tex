\apendice{Especificación de diseño}

\section{Introducción}

Este anexo describe el diseño general del sistema desarrollado.

\section{Diseño arquitectónico}

La arquitectura sigue un modelo basado en servicios cloud desplegados sobre Microsoft Azure.

Los principales componentes son:

\begin{itemize}
    \item Azure App Service.
    \item API Flask.
    \item Azure Storage.
    \item Azure Key Vault.
    \item Azure Monitor.
\end{itemize}

\section{Diseño procedimental}

El flujo principal del sistema es:

\begin{enumerate}
    \item Recepción de la incidencia por API REST o Logic App.
    \item Validación de campos obligatorios.
    \item Clasificación automática mediante análisis de palabras clave.
    \item Persistencia en Blob Storage o fichero local.
    \item Respuesta JSON con identificador, prioridad y confianza de clasificación.
\end{enumerate}

\section{Integración cloud}

\begin{itemize}
    \item \textbf{Terraform}: provisiona Resource Group, App Service, Storage y Key Vault.
    \item \textbf{GitHub Actions}: prueba, valida y despliega la solución.
    \item \textbf{Logic App}: automatiza el registro de incidencias desde un trigger HTTP.
\end{itemize}
